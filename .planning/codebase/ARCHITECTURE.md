# Architecture

**Analysis Date:** 2026-04-14

## Pattern Overview

**Overall:** Sequential pipeline architecture with stateless processing stages

**Key Characteristics:**
- Five-stage linear pipeline: Extract → OCR → Analyze → Topic Model → Dashboard
- Each stage produces independent artifacts that feed the next
- Filesystem as the integration layer (images, OCR output, JSON data)
- Dashboard is a static single-page application consuming pre-generated JSON
- Urdu language specialized processing (stop words, tokenization, NER)

## Layers

**Source Input Layer:**
- Purpose: Accept books in various formats (EPUB, PDF, image folders)
- Location: `books/<slug>/source/`
- Contains: EPUB files, PDFs, or pre-extracted image files
- Depends on: User provides source file via pipeline.py --new
- Used by: `extract_images.py`

**Image Extraction Layer:**
- Purpose: Normalize all source formats to image files for OCR
- Location: `scripts/extract_images.py` and `books/<slug>/images/`
- Contains: Extracted page images (JPEG, PNG, etc.) with sequential naming (page_0000.jpg)
- Depends on: Source files in books/*/source/
- Used by: OCR pipeline
- Process:
  - EPUB: Unzips and extracts embedded images
  - PDF: Converts each page to 300 DPI image (via PyMuPDF/fitz)
  - Images: Direct copy from source/ to images/
  - Skips extraction if images already exist

**OCR Processing Layer:**
- Purpose: Extract text from page images using Google Drive API
- Location: `scripts/ocr_google_drive.py` and `books/<slug>/output/`
- Contains: Plain text files (page_0000.txt, page_0001.txt) with extracted OCR text
- Depends on: Image files, Google Drive API credentials (config/credentials.json, config/token.json)
- Used by: Analysis pipeline
- Process:
  - Uploads each image as a Google Doc (triggers OCR)
  - Downloads extracted text as plain text
  - Deletes temporary files
  - Retries with exponential backoff on failure
  - Skips already-processed pages

**Analysis Layer:**
- Purpose: Extract linguistic features and generate dashboard-ready JSON
- Location: `scripts/analyze.py`
- Outputs: `dashboard/data/`
- Contains:
  - Per-book analysis: `<slug>.json` with word frequency, bigrams, TF-IDF, page stats, search index
  - Index: `index.json` aggregating stats across all books
- Depends on: OCR output files in books/*/output/
- Used by: Dashboard (via app.js)
- Key analyses:
  - **Word frequency**: Top 100 words with counts (stop words filtered)
  - **Tokenization**: Urdu-aware splitting with stop word removal
  - **Page stats**: Words/tokens per page (for trajectory charts)
  - **Bigrams**: Top 50 two-word sequences
  - **Co-occurrence graph**: Word pairs within 5-word sliding window
  - **TF-IDF**: Per-page importance scores (surfaces distinctive words)
  - **Search index**: Full page text with metadata (page_num, image_path)

**Topic Modeling Layer:**
- Purpose: Embed pages, cluster into topics, generate 2D visualization
- Location: `scripts/topic_model.py`
- Outputs: `dashboard/data/topics-<slug>.json`
- Contains:
  - Points: 2D coordinates (UMAP reduction), cluster labels, page snippets
  - Cluster samples: Text samples per topic for labeling
- Depends on: `dashboard/data/<slug>.json` (analysis output)
- Used by: Dashboard "Topics" tab
- Process:
  - Loads search_pages from analysis JSON
  - Encodes pages using multilingual embeddings (sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2)
  - Reduces to 2D with UMAP (n_neighbors=15, metric=cosine)
  - Clusters with HDBSCAN (adaptive min_cluster_size)
  - Outputs cluster membership and visualization coordinates

**Entity & Metadata Layer:**
- Purpose: Manually curated enrichment data for entity relationships and book info
- Location: `dashboard/data/entities-<slug>.json`, `dashboard/data/metadata.json`, `dashboard/data/translations.json`
- Contains:
  - Entities: People, Places, Themes with Urdu/English names and relationships
  - Metadata: Author, category, tags per book
  - Translations: Mapping of terms for UI
- Used by: Dashboard "Relationships" tab
- Not generated by pipeline (human-curated)

**Dashboard Layer:**
- Purpose: Serve interactive static visualization of all analyses
- Location: `dashboard/index.html`, `dashboard/app.js`, `dashboard/data/*.json`
- Contains:
  - Single-page HTML with embedded styles
  - Client-side JavaScript for data loading, filtering, visualization
  - Pre-computed JSON datasets
- Depends on: All JSON files in dashboard/data/
- Used by: Web browser (no backend required)
- Functions:
  - Library view: Browse all books with comparative stats
  - Book detail view: Four tabs (Words, Topics, Relationships, Search)
  - Global search: Full-text search across all books
  - Theme toggle: Light/dark mode

## Data Flow

**End-to-end pipeline invocation:**

```
user: python scripts/pipeline.py <slug>
  ↓
  → [check if images exist]
       if not: extract_images.run(book_dir)
           EPUB/PDF/images → books/<slug>/images/*.jpg
       ↓
  → ocr_google_drive.run(book_dir)
       books/<slug>/images/*.jpg → Google Drive OCR → books/<slug>/output/*.txt
       ↓
user (later): python scripts/analyze.py <slug>
  ↓
  → analyze_book(book_dir)
       books/<slug>/output/*.txt → NLP processing → dashboard/data/<slug>.json
       ↓
  → topic_model.py <slug> (optional)
       dashboard/data/<slug>.json → embeddings → UMAP/HDBSCAN → dashboard/data/topics-<slug>.json
       ↓
user: open dashboard/index.html
  ↓
browser: app.js loads dashboard/data/*.json
  ↓
  → renders Library view (all books)
  → on book click: loads <slug>.json, topics-<slug>.json, entities-<slug>.json
  → renders Book detail with 4 tabs
```

**Per-pipeline-stage data contracts:**

| Stage | Input | Output | Format |
|-------|-------|--------|--------|
| Extract | EPUB/PDF/images | Page images | `*.jpg`, `*.png` |
| OCR | Images | OCR text | Plain text `*.txt` |
| Analyze | OCR text | Book analysis | JSON (word_freq, stats, graph, search_pages, tfidf) |
| Topic Model | Analysis JSON | Cluster coordinates | JSON (points, n_clusters, cluster_samples) |
| Dashboard | All JSONs | HTML + app state | Browser DOM |

## Key Abstractions

**Book Structure:**
- Purpose: Standardized directory layout for all books
- Location: `books/<slug>/` with subdirs: source/, images/, output/
- Pattern: Slug-based identity allows referencing by name throughout pipeline

**Stop Words (Urdu):**
- Purpose: Filter common words before analysis
- Location: `scripts/analyze.py` lines 24-34 (URDU_STOP_WORDS set)
- Usage: Applied in tokenize() and TfidfVectorizer config
- Includes: Common pronouns, conjunctions, auxiliary verbs, prepositions

**Text Cleaning Regex:**
- Purpose: Remove punctuation, diacritics, Latin, digits from Urdu text
- Location: `scripts/analyze.py` lines 36-44 (CLEAN_RE)
- Applied: Before word counting and tokenization
- Preserves: Only core Urdu characters (U+0600-U+06FF)

**Page Extraction:**
- Purpose: Link OCR text files to source images for search results
- Function: `analyze.py:find_image_for_stem()` (lines 54-61)
- Output: Relative paths like `books/ankhain/images/page_0022.jpg`
- Used: In search_pages for displaying page scans in modal

**Entity Graph:**
- Purpose: Represent people, places, themes and their relationships
- Location: `dashboard/data/entities-<slug>.json`
- Structure: Nodes (people/places/themes) + Edges (relationship types: same_person, located_in, theme_of, etc.)
- Usage: Rendered in "Relationships" tab via vis-network.js

## Entry Points

**CLI - Extract Images:**
- Location: `scripts/extract_images.py main()`
- Triggers: User runs `python scripts/extract_images.py <slug>`
- Responsibilities: Resolve book directory, call extract_epub/extract_pdf/copy_images

**CLI - OCR:**
- Location: `scripts/ocr_google_drive.py main()`
- Triggers: User runs `python scripts/ocr_google_drive.py <slug>`
- Responsibilities: Authenticate Google Drive, process all images, save text

**CLI - Analysis:**
- Location: `scripts/analyze.py main()`
- Triggers: User runs `python scripts/analyze.py [slug]`
- Responsibilities: Load OCR output, compute all NLP features, save JSON

**CLI - Topic Modeling:**
- Location: `scripts/topic_model.py main()`
- Triggers: User runs `python scripts/topic_model.py [slug]`
- Responsibilities: Load analysis JSON, embed, cluster, save coordinates

**CLI - Orchestrator:**
- Location: `scripts/pipeline.py main()`
- Triggers: User runs `python scripts/pipeline.py [slug|--new|--list]`
- Responsibilities: 
  - Parse arguments (--extract, --ocr flags)
  - Skip extraction if images exist
  - Call extract_images.run() and ocr_google_drive.run() as needed

**Web - Dashboard:**
- Location: `dashboard/index.html` + `dashboard/app.js`
- Triggers: User opens index.html in browser
- Responsibilities: Load data/index.json, render library, handle navigation/search/filtering

## Error Handling

**Strategy:** Fail-fast with informative messages; allow partial pipeline success

**Patterns:**

- **Missing input validation:** 
  - `pipeline.py`: "Book not found" → list available books
  - `extract_images.py`: "No supported source file found" with list of accepted formats
  - `ocr_google_drive.py`: "No images found" → graceful exit

- **Authentication failures (OCR):**
  - `ocr_google_drive.py:authenticate()`: Check token.json, refresh if expired, run flow if needed
  - Missing credentials.json: Instructions to download from Google Cloud Console

- **Retry logic (OCR):**
  - `ocr_file()`: Exponential backoff (2^n seconds) up to 3 retries
  - Cleans up orphaned Google Drive files on exception

- **Data validation (Analysis):**
  - Skip empty pages in topic modeling (MIN_TEXT_LEN = 40 chars)
  - Gracefully handle books with < 5 pages (skip topic modeling)

- **File existence (Analysis):**
  - Check output_dir exists before iterating
  - Return empty dict if no output files found

## Cross-Cutting Concerns

**Logging:**
- Approach: Print statements to stdout (no logging framework)
- Pattern: Progress indicators "[i/N] filename..."
- Examples: `scripts/ocr_google_drive.py` line 134: `print(f"[{i}/{len(files)}] {f.name}...")`

**Validation:**
- Text cleaning: Regex-based (see CLEAN_RE)
- Stop word filtering: Set membership check
- Urdu tokenization: Simple whitespace split + punctuation stripping

**Authentication (Google Drive):**
- Flow: OAuth2 with InstalledAppFlow (port 0 = automatic free port)
- Token refresh: Automatic if credentials.valid but not expired
- Token storage: `config/token.json` (DO NOT COMMIT)

**Paths:**
- All paths relative to project root via PROJECT_ROOT = Path(__file__).resolve().parent.parent
- Cross-platform: pathlib.Path used throughout (works on Windows/Linux/Mac)
- Book resolution: `resolve_book_dir()` accepts slug or full path

**File Encoding:**
- OCR output: UTF-8 with ensure_ascii=False
- JSON output: UTF-8 with ensure_ascii=False (preserves Urdu script)
