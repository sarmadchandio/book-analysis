## Directory Purposes

**`scripts/`:**
- Purpose: Python pipeline stages (extract → OCR → analyze → topic model)
- Contains: Executable pipeline scripts
- Key files: `pipeline.py` (orchestrator), `extract_images.py`, `ocr_google_drive.py`, `analyze.py`, `topic_model.py`

**`books/`:**
- Purpose: Central storage for all book data (source + processing artifacts)
- Contains: Per-slug subdirectories with source files, images, OCR output
- Pattern: Each book is a self-contained directory with predictable structure

**`books/<slug>/source/`:**
- Purpose: Input file storage (one EPUB, PDF, or folder of images)
- Contains: Original source material for extraction
- Typical files: `book.epub`, `book.pdf`, or `*.jpg`
- Pipeline behavior: Extracted once; skipped if images already exist

**`books/<slug>/images/`:**
- Purpose: Normalized page images for OCR processing
- Contains: Sequential image files named `page_0000.jpg`, `page_0001.jpg`, etc.
- Created by: `extract_images.py` (from EPUB, PDF, or copied from source/)
- Used by: `ocr_google_drive.py`

**`books/<slug>/output/`:**
- Purpose: OCR-extracted text files (one per page)
- Contains: Plain text files named `page_0000.txt`, `page_0001.txt`, etc.
- Created by: `ocr_google_drive.py` (via Google Drive API)
- Used by: `analyze.py`

**`dashboard/`:**
- Purpose: Static web dashboard for visualization
- Contains: Single HTML file, JavaScript, and pre-computed JSON data
- Deployment: Copy entire folder to web server; no backend required

**`dashboard/data/`:**
- Purpose: Pre-generated JSON datasets consumed by dashboard app.js
- Contains:
  - `index.json`: Metadata about all books (name, total_pages, total_words, lexical_richness, etc.)
  - `<slug>.json`: Complete analysis for one book (stats, word_freq, bigrams, page_stats, graph, search_pages, tfidf)
  - `topics-<slug>.json`: Topic clusters in 2D space (points, n_clusters, cluster_samples)
  - `entities-<slug>.json`: Curated NER entities and relationships (manually created)
  - `metadata.json`: Book metadata (author, category, tags)
  - `translations.json`: UI string mappings
- Generated by: `analyze.py` and `topic_model.py` (except entities-*, metadata.json, translations.json which are manual)

**`config/`:**
- Purpose: Application configuration and credentials
- Contains: Google Drive OAuth2 files (credentials.json, token.json)
- Access: Referenced by `ocr_google_drive.py` for authentication

## Naming Conventions

**Files:**

- **Python scripts**: `snake_case.py` (e.g., `extract_images.py`, `ocr_google_drive.py`)
- **JSON data**: `<slug>.json` or `<category>-<slug>.json` (e.g., `ankhain.json`, `topics-ankhain.json`, `entities-ankhain.json`)
- **OCR output**: `page_NNNN.txt` where NNNN is zero-padded page number (e.g., `page_0000.txt`, `page_0135.txt`)
- **Images**: Same pattern as OCR: `page_NNNN.{jpg,png,gif,etc}` (e.g., `page_0000.jpg`)

**Directories:**

- **Book slugs**: `kebab-case` (e.g., `deewan-e-ghalib`, `murda-ankhain`, `ankhain`)
- **Pipeline stages**: Functional names (source, images, output)
- **Python package structure**: Flat (no __init__.py, all scripts at scripts/ root)

**Variables & Constants:**

- **Projects paths**: UPPER_SNAKE_CASE (e.g., `PROJECT_ROOT`, `BOOKS_DIR`, `DASHBOARD_DATA`)
- **Data structures**: Snake_case (e.g., `book_data`, `analysis_results`, `page_stats`)
- **Config sets**: UPPER_SNAKE_CASE (e.g., `URDU_STOP_WORDS`, `IMAGE_EXTENSIONS`)

## Where to Add New Code

**New pipeline stage (e.g., translation, validation):**
- Implementation: `scripts/my_stage.py`
- Entry point: `my_stage.py:run(book_dir: Path)` function
- Integration: Import and call from `pipeline.py` after existing stages
- Data contract: Accept book_dir, read from one of (source/, images/, output/), write to new directory or append to existing
- Example: Post-processing validator consuming books/*/output/ → books/*/validated/

**New analysis feature (e.g., rhyme scheme, named entity recognition):**
- Implementation: Add function to `scripts/analyze.py`
- Location: After existing analyses in `analyze_book()` (lines 106-227)
- Output: Add to the returned dict (alongside stats, word_freq, bigrams, graph, search_pages, tfidf)
- JSON generation: Output field is automatically saved to dashboard/data/<slug>.json
- Dashboard integration: Import in dashboard/app.js, render in appropriate tab
- Example: POS tagging added to analyze_book() would add to book data JSON automatically

**New dashboard visualization:**
- Implementation: 
  - Add canvas/div in dashboard/index.html (in appropriate tab: words/topics/graph/search)
  - Add chart rendering code in dashboard/app.js (function name should start with "render" or "show")
  - Load data from booksData[currentBook] or booksIndex
- Chart library: Chart.js (already included) or vis-network.js (for graph viz)
- Data access pattern: `booksData[slug].word_freq`, `booksData[slug].topics`, `booksData[slug].graph`, etc.
- Theme support: Use CSS variables (--accent, --text-primary, etc.) for light/dark mode

**New metadata field (author, category, tags):**
- File: `dashboard/data/metadata.json`
- Structure: `{ "<slug>": { "author": "...", "category": "...", "tags": [...], "new_field": "..." } }`
- Dashboard integration: Read in `loadBookMeta()` (line 21), use in `getMeta()` (line 24)
- Filter integration: Add to filter dropdowns in `initFilters()` if needed (lines 132-165)

**New curated entity type (beyond people/places/themes):**
- File: `dashboard/data/entities-<slug>.json`
- Structure: Add new array (e.g., "concepts": [{"urdu": "...", "english": "...", ...}])
- Relationships: Add edges with "from": "entity", "to": "new_entity" as needed
- Dashboard: Render in "Relationships" tab (vis-network graph will automatically include new nodes)

## Book Processing Status Tracking

**How to know if a book is ready for the dashboard:**

1. **Has images?** Check if `books/<slug>/images/` contains files
   - If empty: Run `python scripts/pipeline.py <slug> --extract`
2. **Has OCR output?** Check if `books/<slug>/output/` contains .txt files
   - If empty: Run `python scripts/pipeline.py <slug> --ocr`
3. **Has analysis?** Check if `dashboard/data/<slug>.json` exists
   - If missing: Run `python scripts/analyze.py <slug>`
4. **Has topics?** Check if `dashboard/data/topics-<slug>.json` exists
   - If missing: Run `python scripts/topic_model.py <slug>` (optional, for Topics tab)
5. **Has metadata?** Check if book is in `dashboard/data/metadata.json`
   - If missing: Add entry with author, category, tags
6. **Has entities?** Check if `dashboard/data/entities-<slug>.json` exists
   - If missing: Create manually (optional, for Relationships tab)

**Quick status check:**
```bash
# All three books ready?
ls books/ankhain/output/*.txt dashboard/data/ankhain.json dashboard/data/topics-ankhain.json

# Regenerate all analyses
python scripts/analyze.py
python scripts/topic_model.py
```
